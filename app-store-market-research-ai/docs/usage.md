# 使い方

## 基本の流れ

1. **ランキング取得**（`/rankings`）
   国・ジャンル・無料/有料・件数（Top10/25/50）を指定して Apple公式RSS からランキングを取得します。
   取得したアプリはiTunes Lookup APIで詳細（評価・価格など）も自動的に補完されます。

2. **ジャンル分析**（`/analysis/genre`）
   指定ジャンルの上位アプリの共通点、価格傾向、レビュー不満、狙える余白を分析し、
   市場需要・競合過密・不満余地・個人開発向き・AI開発向きの5スコアを算出します。
   推奨アプリ案・作らない方がいいアプリ案も同時に生成されます。

3. **アプリURL診断**（`/diagnosis`）
   任意のApp Store URL（例: `https://apps.apple.com/jp/app/example/id1234567890`）を入力すると、
   同ジャンル上位アプリと比較した強み・弱み・改善点・診断スコアを表示します。

4. **レビュー分析**（`/reviews`）
   取得可能な範囲の公開レビュー、またはお手元のレビューCSVをインポートして分析します。
   カテゴリ別に不満・要望を集計し、新規アプリ案のヒントを表示します。

5. **アプリ案生成**（`/ideas`）
   ジャンル分析結果（analysisId）から、Claude Code / Codex 向けの開発プロンプトを含む
   アプリ案を確認・コピーできます。

6. **レポート**（`/reports`）
   生成したレポートをMarkdownで再表示・コピーできます。

## レビューCSVの形式

| 列 | 必須 | 内容 |
| --- | --- | --- |
| appId | 任意（アップロード時のappIdが優先） | アプリID |
| rating | 必須 | 星評価（1〜5） |
| title | 任意 | レビュータイトル |
| body | 必須 | レビュー本文 |
| author | 任意 | 投稿者 |
| createdAt | 任意 | 投稿日 |
| country | 任意 | 国 |
| lang | 任意 | 言語 |

## 注意事項

- App Store画面のHTMLスクレイピングは行いません。ランキングはApple公式RSS、アプリ詳細は
  iTunes Search/Lookup APIのみを使用します。
- レビュー本文の大量取得・競合レビューの網羅的な取得は保証していません。
- スコアやアプリ案は分析結果であり、成功・収益を保証するものではありません。
